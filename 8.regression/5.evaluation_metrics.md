# 11.5 Evaluation Metrics — Regression

---

### 📖 Overview
> Regression doesn't use accuracy — predictions are continuous numbers, so we measure **how far off** predictions are from actuals. The four key metrics are MAE, MSE, RMSE, and R².

---

## 1. MAE — Mean Absolute Error

### 📖 Definition
> Average of the **absolute differences** between actual and predicted values. Every error counts equally regardless of size.

```
MAE = (1/n) × Σ |y - ŷ|
```

### 🧑‍🏫 Layman Explanation
If you predict house prices and your errors are +10k, -5k, +10k — MAE just averages their absolute sizes: (10 + 5 + 10) / 3 = 8.33. It tells you: "on average, you're off by 8.33 units." Simple and honest — a ₹100k error counts 10x more than a ₹10k error, nothing sneaky.

### 💡 Example from PDF
```
Actual:    100, 120, 140
Predicted: 110, 115, 130
Errors:     10,   5,  10   ← absolute differences

MAE = (10 + 5 + 10) / 3 = 8.33
Meaning: on average, prediction is 8.33 units away from actual
```

### ⭐ Key Highlights
> 🔑 **Treats all errors equally** — a 100-unit error counts 10× a 10-unit error (linear)
> 🔑 **Robust to outliers** compared to MSE
> 🔑 Use when you want **simple, interpretable, robust** error measurement

---

## 2. MSE — Mean Squared Error

### 📖 Definition
> Average of the **squared differences** between actual and predicted values. Large errors are penalized disproportionately.

```
MSE = (1/n) × Σ (y - ŷ)²
```

### 🧑‍🏫 Layman Explanation
Same setup, but now square each error before averaging. That 100-unit error becomes 10,000 — one massive mistake dominates. MSE screams about big errors. This is intentional: if huge mistakes are dangerous (financial forecasting, medical dosing), MSE is the right measure to optimize.

### 💡 Example from PDF
```
Errors:          10,   5,  10
Squared errors: 100,  25, 100

MSE = (100 + 25 + 100) / 3 = 75

Note: MSE is in units² (not the same unit as y) → use RMSE to interpret
```

### ⭐ Key Highlights
> 🔑 **Penalizes large errors heavily** (squared) — outlier-sensitive
> 🔑 **Units are squared** → harder to interpret directly (use RMSE for interpretation)
> 🔑 MSE is the default **loss function for training** regression models (gradient descent minimizes MSE)
> 🔑 Use when large errors are especially costly

---

## 3. RMSE — Root Mean Squared Error

### 📖 Definition
> Square root of MSE. Returns error in the **same units as the target variable**, making it interpretable like MAE but still penalizing large errors.

```
RMSE = √MSE
```

### 🧑‍🏫 Layman Explanation
MSE was in "squared rupees" — confusing. RMSE just takes the square root to bring it back to "rupees." Now you can say: "on average, my predictions deviate by ₹8.66 lakh." Same penalty structure as MSE (large errors hurt more), but readable like MAE.

### 💡 Example from PDF
```
MSE = 75
RMSE = √75 ≈ 8.66

Meaning: predictions deviate by ~8.66 units from actuals on average
```

### ⭐ Key Highlights
> 🔑 **Same units as target variable** → directly interpretable
> 🔑 **Penalizes large errors more** than MAE (inherited from MSE)
> 🔑 **RMSE vs MAE**: if RMSE >> MAE, you likely have large outlier errors
> 🔑 Most common metric to report for regression

---

## 4. R² Score — Coefficient of Determination

### 📖 Definition
> R² measures **how much of the variance in the target variable is explained by the model** — relative to a baseline that just predicts the mean every time.

```
R² = 1 - [ Σ(y - ŷ)² / Σ(y - ȳ)² ]

Where ȳ = mean of actual values
```

### 🧑‍🏫 Layman Explanation
The worst possible model just predicts the mean for every data point. R² asks: "Compared to that do-nothing baseline, how much better are we?" R² = 1.0 means perfect fit. R² = 0 means we're no better than predicting the mean. R² = 0.85 means the model explains 85% of why prices vary — 15% is still unexplained.

### 💡 Example from PDF
```
R² = 0.85
Meaning: the model explains 85% of the variance in the target variable
         15% of variation is still unexplained (noise, missing features, etc.)

Interpretation guide:
  R² = 1.00  → Perfect (suspicious — may be overfitting)
  R² = 0.85+ → Excellent
  R² = 0.70  → Good
  R² = 0.50  → Moderate
  R² = 0.00  → No better than predicting the mean
  R² < 0.00  → Worse than predicting the mean (model is broken)
```

### ⭐ Key Highlights
> 🔑 **Relative metric** — always compare model vs "predict the mean" baseline
> 🔑 Range is typically 0 to 1 (can go negative for terrible models)
> 🔑 ⚠️ **High R² alone is not enough** — always also check MAE/RMSE for absolute error magnitude
> 🔑 Adding more features always increases R² (use **Adjusted R²** for multi-feature models)

---

## 💡 All Metrics — Code Together

```python
from sklearn.metrics import (
    mean_absolute_error, mean_squared_error, r2_score
)
import numpy as np

y_actual    = [100, 120, 140, 160, 180]
y_predicted = [110, 115, 130, 165, 175]

mae  = mean_absolute_error(y_actual, y_predicted)
mse  = mean_squared_error(y_actual, y_predicted)
rmse = np.sqrt(mse)
r2   = r2_score(y_actual, y_predicted)

print(f"MAE:  {mae:.2f}   ← avg absolute error")
print(f"MSE:  {mse:.2f}   ← avg squared error (units²)")
print(f"RMSE: {rmse:.2f}  ← avg error in original units")
print(f"R²:   {r2:.4f}  ← % variance explained by model")
```

## ⚡ Metrics Quick-Reference Card

| Metric   | Formula| Sensitive to Outliers? | Units     | Use When |
|-------- -|--------------------|------------|-----------|---------------------|
| **MAE**  | Mean of `|y - ŷ|`  | ❌ Low     | Same as y | Robust, interpretable baseline |
| **MSE**  | Mean of `(y - ŷ)²` | ✅ High    | Squared y | Training loss, large errors costly |
| **RMSE** | `√MSE`             | ✅ High    | Same as y | Reporting; large errors costly |
| **R²**   | `1 - SS_res/SS_tot`| —          | 0 to 1 | How well model explains variance |

> ⚠️ **Always report at least RMSE + R² together.** R² tells you relative fit; RMSE tells you absolute error magnitude.
