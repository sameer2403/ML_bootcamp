# 11.2 Regression Models — Intuition
## Models: Linear Regression, Lasso (L1) & Ridge (L2)

---

## 1. Linear Regression

### 📖 Definition
> **Linear Regression** fits a **straight line** (or hyperplane in multiple dimensions) through the data by learning weights `w` and bias `b` such that the **sum of squared errors between predictions and actuals is minimized**.

---

### 🧑‍🏫 Layman Explanation
You want to predict house price from size. Plot size on X-axis, price on Y-axis. Linear regression draws the best-fitting straight line through all the dots — not through every point exactly, but as close as possible to all of them on average. Then to predict price for a new house, you just read off the line.

---

### ⚙️ Technical Explanation

**The equation:**
```
ŷ = w₁x₁ + w₂x₂ + ... + wₙxₙ + b

Where:
  ŷ   = predicted value
  w   = weights (learned — how much each feature matters)
  x   = input features
  b   = bias (shifts the line up or down)
```

**How it works — 3 steps:**

**Step 1 — Compute Prediction:** `ŷ = Wx + b` (weighted sum of inputs)

**Step 2 — Measure Error:** Residual = `y - ŷ` (actual minus predicted)

**Step 3 — Minimize Error:** Use **Ordinary Least Squares (OLS)** or **Gradient Descent** to find the `w` and `b` values that minimize the total squared error (MSE):
```
MSE = (1/n) Σ (y - ŷ)²
```

```
y (price)
│                     /  ← best-fit line (y = wx + b)
│               *   /
│          *      /
│     *        /
│         *  /
│       *  /
│     * /
└──────────────→ x (size)
```

---

### 💡 Code

```python
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_absolute_error, r2_score
import numpy as np

# Simple example: predict salary from years of experience
np.random.seed(42)
X = np.random.rand(100, 1) * 10        # Years of experience (0-10)
y = 3 * X.squeeze() + 25 + np.random.randn(100) * 2  # Salary in lakhs

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

model = LinearRegression()
model.fit(X_train, y_train)

y_pred = model.predict(X_test)

print(f"Weight (slope):  {model.coef_[0]:.2f}")   # ~3.0 → ₹3L per year of exp
print(f"Bias (intercept):{model.intercept_:.2f}")  # ~25 → base salary
print(f"MAE:  {mean_absolute_error(y_test, y_pred):.2f}")
print(f"R²:   {r2_score(y_test, y_pred):.4f}")

# Interpret weights: positive w → feature increases prediction
#                   negative w → feature decreases prediction
```

---

### ✅ When to Use / ❌ Limitations

| ✅ When to Use                  | ❌ Limitations |
|--------------------------------|---------------------------------|
| Clear **linear trend** in data | **Fails on non-linear data** |
| Need **simple baseline** model | **Sensitive to outliers** (squared error amplifies them) |
| **Interpretability** matters (weights = feature effect) | Assumes **no multicollinearity** between features |

---

### ⭐ Key Highlights
> 🔑 Learns `w` (how much each feature matters) and `b` (baseline offset)
> 🔑 **Minimizes Mean Squared Error** using OLS or Gradient Descent
> 🔑 Always start with Linear Regression as your **regression baseline**

## 2. Lasso Regression (L1 Regularization)

### 📖 Definition
> **Lasso** (Least Absolute Shrinkage and Selection Operator) adds an **L1 penalty** (sum of absolute values of weights) to the loss function. This forces some weights to become **exactly zero**, effectively performing **automatic feature selection**.

---

### 🧑‍🏫 Layman Explanation
You have 100 features to predict house price. Most are noise (day of week you visited, agent's name, etc.). Lasso acts like a ruthless editor: it keeps only the features that truly matter and sets the rest to zero. It's Linear Regression with a "budget" — spend your weight budget only on what matters, force the rest to nothing.

---

### ⚙️ Technical Explanation

**Loss function (Lasso):**
```
Loss = MSE + λ × Σ|wᵢ|
         ↑ fit   ↑ L1 penalty
```
- `λ` (alpha) controls how strong the penalty is
- Higher `λ` → more weights pushed to zero → sparser model (**only important features**)
- The L1 penalty has "sharp corners" in the geometry that cause weights to land at exactly zero (unlike L2)

**How it works:**
1. Fit same linear equation `ŷ = Wx + b`
2. Apply L1 penalty — pushes weak feature weights to **exactly 0**
3. Result: **sparse model** with only the most important features retained

```python
from sklearn.linear_model import Lasso
import numpy as np

X_train_m = np.random.randn(200, 20)  # 20 features, most irrelevant
y_train_m  = 3*X_train_m[:,0] + 2*X_train_m[:,1] + np.random.randn(200)

lasso = Lasso(alpha=0.1)   # alpha = λ, regularization strength
lasso.fit(X_train_m, y_train_m)

# Many weights will be exactly 0 — those features are "removed"
print("Lasso Weights:", lasso.coef_)
nonzero = (lasso.coef_ != 0).sum()
print(f"Features kept: {nonzero} / 20")  # Lasso automatically selects

# Use LassoCV to auto-find the best alpha
from sklearn.linear_model import LassoCV
lasso_cv = LassoCV(cv=5).fit(X_train_m, y_train_m)
print(f"Best alpha: {lasso_cv.alpha_:.4f}")
```

---

### ✅ When to Use / ❌ Limitations

| ✅ When to Use                        | ❌ Limitations                        |
|---------------------------------------|---------------------------------------|
| **Many unnecessary / noisy features** | **Unstable with correlated features** — may drop one of two correlated features arbitrarily |
| Need **automatic feature selection** | May drop useful variables if alpha is too high |
| **Sparse model** preferred (few active features) | |

---

### ⭐ Key Highlights
> 🔑 **L1 penalty = Σ|wᵢ|** → pushes weights to exactly **zero** → feature selection
> 🔑 `alpha` controls sparsity: higher alpha → more zeros → simpler model
> 🔑 Use **LassoCV** to find the optimal alpha automatically
> 🔑 Lasso = "Linear Regression with a zero-setter"

---

## 3. Ridge Regression (L2 Regularization)

### 📖 Definition
> **Ridge** adds an **L2 penalty** (sum of squared weights) to the loss function. Unlike Lasso, it **shrinks weights toward zero but never to exactly zero** — it keeps all features but prevents any single one from dominating.

---

### 🧑‍🏫 Layman Explanation
Imagine you have 100 features and many are correlated (e.g., "house size in sqft" and "house size in sq metres"). Linear regression might give huge weights to both, making the model unstable. Ridge says: "spread the weight more evenly, don't let any one feature get too big." It dials down everyone slightly — nobody gets eliminated, but nobody gets outsized influence either.

---

### ⚙️ Technical Explanation

**Loss function (Ridge):**
```
Loss = MSE + λ × Σwᵢ²
         ↑ fit   ↑ L2 penalty (sum of squared weights)
```
- Penalizes large weights (not zero, just small)
- The L2 penalty has a **circular** geometry — weights shrink toward 0 but rarely reach it
- Result: **Bias slightly increases, Variance decreases** → better generalization

|                    | Lasso (L1)                 | Ridge (L2)         |
|--------------------|----------------------------|--------------------|
| Penalty            | `Σ|wᵢ|`                    | `Σwᵢ²` |
| Effect on weights  | Sets some to **exactly 0** | Shrinks all, none exactly 0 |
| Feature selection? | ✅ Yes                     | ❌ No (keeps all features) |
| Best for           | Many irrelevant features   | Multicollinearity |

```python
from sklearn.linear_model import Ridge, RidgeCV

ridge = Ridge(alpha=1.0)   # alpha = λ
ridge.fit(X_train_m, y_train_m)

print("Ridge Weights:", ridge.coef_)
# All weights are non-zero but smaller than plain Linear Regression

# Auto-find best alpha
ridge_cv = RidgeCV(alphas=[0.01, 0.1, 1.0, 10.0], cv=5)
ridge_cv.fit(X_train_m, y_train_m)
print(f"Best alpha: {ridge_cv.alpha_}")
```

---

### ✅ When to Use / ❌ Limitations

| ✅ When to Use                                      | ❌ Limitations                                      |
|-----------------------------------------------------|-----------------------------------------------------|
| **Multicollinearity present** (correlated features) | **Does not remove features** (can't do feature selection) |
| **Many features**, want to keep all                 | Harder to interpret than plain Linear Regression |
| Want **stable model** that doesn't overfit          | |

---

### ⭐ Key Highlights
> 🔑 **L2 penalty = Σwᵢ²** → shrinks weights toward zero, **never exactly zero**
> 🔑 **Ridge handles multicollinearity** — Lasso cannot
> 🔑 `Bias ↑ slightly, Variance ↓ significantly` → better test performance
> 🔑 Use **RidgeCV** to auto-tune alpha

## 🧠 Brain Map — Linear / Lasso / Ridge

```
           REGRESSION MODELS (Linear Family)
                        │
          ┌─────────────┼─────────────┐
          │             │             │
     Linear Reg      Lasso (L1)   Ridge (L2)
          │             │             │
    y = Wx + b    + λΣ|w|       + λΣw²
    Minimizes MSE  Some w → 0   All w shrink
    No penalty     Feature sel   Handles collin.
          │             │             │
    Use when:     Use when:      Use when:
    Linear data   Many noisy     Correlated
    Baseline      features       features
```

## ⚡ Quick Comparison: Linear vs Lasso vs Ridge

|                           | Linear                    | Lasso (L1)               | Ridge (L2) |
|---------------------------|---------------------------|--------------------- ----|---------------------------|
| Penalty                   | None                      | `Σ|wᵢ|`                  | `Σwᵢ²` |
| Feature selection?        | ❌                        | ✅ (exact zeros)         | ❌ |
| Handles multicollinearity? | ❌                       | ❌                       | ✅ |
| Best when                 | Clean, linear data        | Many irrelevant features | Correlated features |

> **Elastic Net** = Lasso + Ridge combined: `λ₁Σ|wᵢ| + λ₂Σwᵢ²` → best of both worlds
