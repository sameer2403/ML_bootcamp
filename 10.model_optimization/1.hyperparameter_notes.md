## 13.1 Hyperparameter Tuning — Intuition

### Definition
- **Model Parameters**: values the model *learns from data* during training (e.g., weight `w`, bias `b`).
- **Hyperparameters**: values *set before training begins* that control *how* the learning happens (e.g., learning rate, number of iterations, number of trees).
- **Hyperparameter Tuning**: the process of searching for the hyperparameter values that give the best model performance.

### Layman Explanation
Think of baking a cake.
- **Model parameters** = the final taste/texture of the cake — this comes out of the process (learned).
- **Hyperparameters** = oven temperature and baking time — you *set these before* you start baking, and they control how well the cake turns out.
- **Tuning** = trying different oven-temperature/time combinations until you get the perfectly baked cake.

### Technical Explanation
**Model Parameters**
- Learned from data
- Updated during training (e.g., via gradient descent)
- Example: for `y = wx + b`, after training you get `w = 2`, `b = 3` → `y = 2x + 3`

**Hyperparameters**
- Fixed *before* the training process starts
- Control *how* learning happens (not what is learned)
- Examples: `learning_rate = 0.5`, `n_iterations = 10`, `n_estimators` (trees in Random Forest), `max_depth`, etc.
  - `lr` (learning rate) → amount of change applied per update step
  - `n` (iterations) → how many times the model updates itself

**Why Tuning is needed**
- Different hyperparameter values → different results.
- Some values cause the model to **overfit** (too flexible), others cause it to **learn poorly / underfit** (too rigid).

**Search strategies:**

| Method             | How it works             | Speed             | Cost             |
|--------------------|--------------------|--------------------|--------------------|
| **Grid Search CV** | Tries **all** combinations of specified hyperparameter values | Slower | Computationally expensive |
| **Randomized Search CV** | Tries a **random** subset of combinations | Faster | Computationally cheaper |

Visually: imagine a grid where one axis is `max_depth` and the other is `n_estimators`.
- Grid Search checks **every intersection point** on the grid.
- Randomized Search checks only a **random sample** of points — much cheaper, and often finds a "good enough" (or even the best) point without exhaustively checking everything.

### Example
Random Forest Classifier hyperparameters to tune: `n_estimators` (number of trees) and `max_depth` (tree depth).
- Grid Search over `n_estimators = [50, 100, 150, 200]` × `max_depth = [3, 5, 7, 10]` → **16 combinations**, all tested.
- Randomized Search with `n_iter=6` → only **6 random combinations** tested, picks the best among those.

### Highlights 🔑
- **Model training** = the process of finding the best **model parameters** (e.g., weights).
- **Hyperparameters decide *how* we reach those best model parameters** — they are the "settings" of the learning process itself.
- Grid Search = thorough but slow; Randomized Search = fast but samples only a subset of the space.
- Tuning is essential because the "right" hyperparameters directly prevent overfitting/underfitting (ties back to Section 12.6).

### Brain Map 🧠
```
PARAMETERS        →  learned FROM data      →  e.g., weight, bias        →  updated during training
HYPERPARAMETERS   →  set BEFORE training    →  e.g., lr, n_iter, depth   →  control HOW learning happens

TUNING METHODS
   ├── Grid Search CV        → ALL combos    → slow, expensive, thorough
   └── Randomized Search CV  → RANDOM combos → fast, cheap, approximate

Mnemonic: "Parameters are learned, Hyperparameters are chosen."
```

### Code
```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV, RandomizedSearchCV

param_grid = {
    "n_estimators": [50, 100, 150, 200],
    "max_depth": [3, 5, 7, 10]
}

# --- Grid Search: tries ALL combinations ---
grid_search = GridSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    param_grid=param_grid,
    cv=5,
    scoring="f1"
)
grid_search.fit(X_train, y_train)
print("Best params (Grid):", grid_search.best_params_)

# --- Randomized Search: tries a RANDOM subset of combinations ---
random_search = RandomizedSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    param_distributions=param_grid,
    n_iter=6,          # only 6 random combos tested
    cv=5,
    scoring="f1",
    random_state=42
)
random_search.fit(X_train, y_train)
print("Best params (Random):", random_search.best_params_)
```
